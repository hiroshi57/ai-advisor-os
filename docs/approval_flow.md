# 一次回答ボット 顧問承認フロー 仕様書 (draft v0.1)

> ステータス: **仕様書ドラフト（承認待ち）** / 対象: B2 ai-advisor-os / 作成: 2026-07-09

## 0. 差別化の核

ボットはクライアントへ**直接返信しない**。顧問コンソールに回答案を提示し、顧問が承認して初めて送信される。
承認済み回答はナレッジベースに蓄積され、運用とともに精度が上がる。

## 1. 状態遷移

```
[質問受信]
   │  RAG検索(カルテ+ナレッジ)
   ▼
(draft) ── 出典あり? ──no──▶ (no_answer) 「ドラフト不可」を顧問に通知（創作しない）
   │yes
   ▼
(pending_review) ── 顧問操作 ──┬─ 承認 ─▶ (sent) → ナレッジへ蓄積
                               ├─ 編集して送信 ─▶ (sent, edited)
                               └─ 自分で対応 ─▶ (handled_by_advisor)
```

## 2. 不変条件（受け入れ基準）

- **顧問未承認の回答はクライアントに送信されない**（テスト担保）。
- ドラフトには必ず出典（カルテ / ナレッジ記事ID）を付与。**出典がなければドラフトを出さない**。
- 全 LLM 出力に承認ログ（誰が・いつ・承認/編集したか）を残す。
